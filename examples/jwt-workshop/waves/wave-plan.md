# JWT orchestration wave

## Wave goal

Deliver a small, reviewable authentication slice with clear planning and verification boundaries.

## Suggested wave structure

1. Planning wave — confirm the living spec and acceptance criteria.
2. Implementation wave — add or sketch login and protected-profile behavior.
3. Verification wave — confirm success and failure cases with explicit evidence.

## Evidence to capture

- spec updates
- intended files or changed files
- test or verification notes
- deferred follow-ups such as refresh tokens or signup
