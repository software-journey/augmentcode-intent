# JWT isolated workspace exercise

## Goal

Practice containing an auth change in one isolated workspace or worktree.

## Exercise

1. Create an isolated branch or worktree for the login-and-profile slice.
2. Keep the scope limited to the spec in `../specs/living-spec.md`.
3. Record which auth concerns you intentionally defer.
4. Plan how you would validate the slice before asking for review.

## Reflection prompts

- What became easier because the auth work was isolated?
- Which files or concerns would be risky to mix into the same change?
- What would you want a verifier to double-check?
