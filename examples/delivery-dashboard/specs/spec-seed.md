# Delivery dashboard spec seed

## Next exercise

Add a shipment detail drawer that opens from the delayed deliveries list.

## Why this is a good follow-up

It extends the operator workflow naturally and lets learners practice state management, scope boundaries, and PR storytelling for a UI slice.

## Seed questions

- What information is essential in the drawer?
- Should the drawer reuse existing shipment summary data or fetch more?
- How will loading, empty, and error states be represented?

## Done when

- the UI behavior is specified before implementation
- the data dependencies are named clearly
- verification steps are documented up front
