# Delivery dashboard isolated workspace exercise

## Goal

Practice isolating one dashboard slice so the change stays easy to review.

## Exercise

1. Create an isolated branch or worktree for the delayed-deliveries slice.
2. Keep the scope anchored to `../specs/living-spec.md`.
3. Note which dashboard ideas are intentionally out of scope.
4. Plan how you would verify the filter, summary cards, and empty state before review.

## Reflection prompts

- Which UI states deserve special attention in isolation?
- What makes dashboard work especially prone to scope creep?
- What would a verifier need to see to trust the slice?
