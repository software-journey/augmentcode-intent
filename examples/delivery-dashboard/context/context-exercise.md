# Delivery dashboard context exercise

## Objective

Identify the exact dashboard areas you would inspect before editing a delayed-deliveries feature.

## Likely context targets

- dashboard route or page component
- shipment list and summary card components
- data loading hooks, stores, or API adapters
- tests covering filters, empty states, and loading behavior

## Questions to answer

- Where is shipment status normalized for display?
- How is filter state currently stored or derived?
- What user-visible language should represent a delayed shipment?

## Expected outcome

You can point to the likely edit targets and explain how they affect the learner's implementation plan.
