# Delivery dashboard orchestration wave

## Wave goal

Deliver one small dashboard slice that improves operator visibility and remains easy to review.

## Suggested wave structure

1. Planning wave — confirm the delayed-deliveries spec and states.
2. Implementation wave — add summary cards and delayed filtering behavior.
3. Verification wave — confirm visible states, filtering behavior, and follow-up boundaries.

## Evidence to capture

- spec updates
- intended files or changed files
- screenshots or verification notes
- deferred follow-ups such as detail drawers or live updates
